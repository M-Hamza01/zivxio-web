Every app we build starts with the same question: what's the one job this needs to do well?

It's tempting to keep adding features — a settings page here, an extra dashboard there. Most of it ends up unused, and it makes the core experience slower and more confusing.

## Minimal isn't the same as basic

A minimal app can still feel premium. The difference is intentionality: every screen, button, and piece of copy earns its place, rather than existing because it was easy to add.

## How this shows up in practice

- We ship the smallest version that's genuinely useful, then improve it based on real usage — not guesses.
- We say no to features that don't directly serve the app's core purpose, even when they'd be easy to build.
- We treat performance and load time as a feature, not an afterthought.

This approach is slower in the short term. It also means the apps we ship tend to stay useful — and used — for longer.
